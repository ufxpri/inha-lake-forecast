1. 프로젝트 개요 (Project Overview)

1.1 배경 및 목적

학교의 랜드마크인 '인경호'는 계절과 시간에 따라 아름다움이 극명하게 갈리지만, 학생들은 직접 가보지 않으면 현재 상태를 알 수 없음.
본 서비스는 **"지금 인경호에 가야 할까?"**라는 질문에 대해 **'뷰티 스코어(0~100점)'**라는 정량적 지표와 **'생성형 AI 기반의 직관적 일러스트'**를 통해 즉각적인 의사결정을 돕고, 캠퍼스 라이프의 질을 높이는 것을 목적으로 함.

1.2 핵심 가치 (Key Value Proposition)

Instant Decision: 복잡한 날씨 정보 대신, 하나의 점수(Score)로 방문 여부 판단.

Visual Experience: 생성형 AI가 그려주는 감성적인 실시간 호수 뷰 제공.

Data Archive: 유저 참여를 통해 호수의 사계절 데이터를 축적하고 AI 모델 고도화.

2. 타겟 유저 및 시나리오 (User Persona & Journey Map)

2.1 메인 타겟 (Primary Target)

인스타그래머 A씨 (21세, 시각디자인과)

특성: 예쁜 풍경 사진 찍기를 좋아하며, SNS 업로드에 민감함.

Pain Point: 무거운 카메라를 들고 나갔는데 날씨가 흐려 허탕 치는 것을 싫어함.

캠퍼스 커플 B씨 (23세, 기계공학과)

특성: 공강 시간에 여자친구와 산책할 장소를 찾음.

Pain Point: 분위기를 잡으러 갔는데 벤치에 앉을 수 없을 정도로 덥거나 추운 상황을 피하고 싶음.

2.2 유저 시나리오 (User Journey)

[Scenario 1: 실패 없는 인생샷]

Trigger: 오후 5시, 창밖 하늘 색깔이 묘하게 예쁨.

Action: 웹페이지 접속 (즐겨찾기/홈 화면 아이콘).

Check:

Zone A: AI가 생성한 '현재 날씨가 반영된 몽환적인 인경호' 그림 확인.

Score: "현재 뷰티 스코어 92점 (매직 아워)" 확인.

Zone B: 그래프를 보니 30분 뒤 점수 하락 예상.

Decision: "지금 당장 나가야겠다!" → 카메라를 챙겨 호수로 이동.

[Scenario 2: 미래 데이트 계획]

Trigger: 다음 주 수요일 공강 시간에 데이트 약속을 잡음.

Action: 웹페이지 접속 → Zone C (캘린더) 확인.

Check: 다음 주 수요일 아이콘이 '흐림/낮은 점수'로 표시됨 (통계 기반 예측).

Decision: "수요일은 호수 대신 실내 카페로 가야겠다." (헛걸음 방지)

3.2 뷰티 스코어 알고리즘 (Beauty Score Logic)

Range: 0 ~ 100점 (정수)
v1.0 산출 공식:

Score = (하늘 상태 가중치 × 40) + (기온 쾌적도 × 30) + (미세먼지 × 20) + (시간대 보정 × 10)

Penalty: 강수 확률 60% 이상(-50), 미세먼지 '매우 나쁨'(-30).

Bonus: 일몰 전후 30분(Golden Hour) +10점.

4. 시스템 아키텍처 및 기술 전략 (Technical Strategy)

4.1 데이터 파이프라인 (Data Pipeline)

"Pre-generation & Caching" 전략으로 비용 절감 및 속도 향상.

Source: 기상청 API, 한국환경공단.

Scheduler (Airflow):

DAG 1 (Weather): 매시 정각 실행 → 기상 데이터 수집 → 스코어 계산 → DB/Redis 갱신.

DAG 2 (Gen AI): 날씨 상태 변경 시 실행 → Stable Diffusion API 호출 → 이미지 S3 업로드 → URL 갱신.

Client: 자사 Backend API를 통해 Redis에 캐싱된 JSON 데이터와 이미지 URL만 조회.

4.2 AI/Design 리소스 전략

Tool: Stable Diffusion (Self-hosted) 또는 OpenAI DALL-E API.

Cost Management: 유저 요청마다 생성하는 것이 아니라, "날씨 상태(맑음/흐림/비) x 시간대(낮/밤/노을)" 조합에 따라 서버가 주기적으로 1장만 생성하여 공용으로 사용.

4.3 개발 스택 (Tech Stack)

Frontend: React/Vue 기반 PWA (Progressive Web App)

배포: Vercel 또는 AWS S3 + CloudFront (CDN).

특징: "홈 화면에 추가" 유도, 모바일 앱과 유사한 UX 제공.

Backend: Python FastAPI

특징: 비동기 처리 우수, AI 라이브러리 호환성 최상.

DB: PostgreSQL (메타데이터), Redis (실시간 스코어 캐싱).

Storage: AWS S3 (유저 업로드 사진 및 AI 생성 이미지 저장).

DevOps: GitHub Actions (CI/CD), Airflow (Scheduling).

5. 향후 로드맵 (Roadmap)

Phase 1 (MVP): 뷰티 스코어 산출, AI 이미지 캐싱 시스템, PWA 웹 배포, 기본 업로드 구현.

Phase 2: 유저 업로드 사진 기반 스코어 알고리즘 Fine-tuning, Web Push Notification 도입 (점수 90점 이상 시).

Phase 3: 교내 상권(카페/식당) 광고 연동 및 커뮤니티 기능 강화.
